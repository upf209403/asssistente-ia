# Pensamento Computacional

## Definição
- Pensamento computacional é o processo de transformar problemas e tarefas do mundo real em passo lógicos executáveis por computadores.

## Ideia central
- Computadores não possuem intuição.
- Computadores precisam de instruções explícitas e não-ambíguas para funcionarem
- Humanos precisam "traduzir" o conhecimento implícito em regras claras.

## Componentes do pensamento computacional
- Decomposição
- Reconhecimento de padrões
- Abstração
- Algoritmos

### Decomposição
#### Definição
- Consiste em quebrar problemas em partes menores.

#### Objetivos
- Reduzir a complexidade.
- Facilitar a análise.
- Permitir resolver cada parte individualmente.

#### Exemplo
Problema:
- Fazer um bolo

Subproblemas:
- Quebrar ovos
- Misturar ingredientes
- Pré-aquecer forno
- Assar

#### Aplicações
- Programação e ciência da computação
- Ciências
- Engenharia
- Resolução de problemas em geral

#### Importância
- Problemas grandes tendem a ser difíceis de resolver diretamente.
- Dividir problemas permite trabalhar com partes menores antes de construir a solução completa. 

### Reconhecimento de Padrões
#### Definição
- Reconhecimento de padrões consiste em identificar semelhanças, repetições, analogias e estruturas recorrentes entre problemas.

#### Objetivos
- Evitar resolver o mesmo problema várias vezes.
- Reutilizar soluções existentes.
- Simplificar raciocínio.

#### Exemplos
- Perceber que vários programas utilizam estruturas de repetição.
- Identificar que diferentes problemas utilizam listas ou vetores.
- Reconhecer fórmulas ou estruturas repetidas.

#### Importância
- Identificar padrões permite construir soluções mais rápidas e reutilizáveis.
- É um compontente fundamental do pensamento científico em geral.

### Abstração
#### Definição
- Abstração consiste em focar apenas nas informações relevantes para resolver um problema.

#### Objetivos
- Ignorar detalhes desnecessários.
- Reduzir complexidade.
- Construir modelos simplificados.

#### Exemplo
Ao programar um sistema bancário:

Informações relevantes:

- Saldo
- Transferências
- Contas

Informações irrelevantes:

- Cor da parede da agência
- Temperatura ambiente

#### Importância
- Sem abstração, problemas reais costumam possuir detalhes demais para serem resolvidos eficientemente.

### Algoritmos
#### Definição
- Algoritmos são sequências finitas, ordenadas e não ambíguas de instruções utilizadas para resolver problemas.

#### Características principais
- Não-ambiguidade: cada passo deve possuir apenas uma interpretação lógica possível.
- Ordenação: alterar a ordem das instruções altera completamente o resultado.
- Finitude: um algoritmo precisa ter um número finito de passos.

#### Exemplo
Não-ambiguidade:
- Como discutido anteriormente, computadores não possuem conhecimento intuitivo.
- O que parece óbvio para seres humanos precisa ser claramente descrito para computadores de modo lógico.

Ordenação:
- Misturar ingredientes
- Assar

produz resultado diferente de

- Assar
- Misturar ingredientes.

Finitude
- Todo algoritmo precisa terminar após um número limitado de etapas.
Um algoritmo infinito não produz resultados.

#### Analogia
- Um algoritmo funciona como uma receita: existe uma sequência definida de passos necessária para produzir determinado resultado.

### Relação entre os componentes
- Os componentes do pensamento computacional não funcionam isoladamente.

#### Fluxo simplificado
- Decomposição divide um problema em partes menores.
- Reconhecimento de padrões identifica semelhanças entre as partes e os subproblemas.
- A abstração filtra apenas as informações relevantes.
- Algoritmos transformam as decisões em uma sequência de passos executáveis por um computador.

#### Exemplo:
Problema:
- Criar um aplicativo bancário.

Decomposição:
- Cadastro
- Saldo
- Transferências
- Segurança

Reconhecimento de padrões:
- Os diferentes módulos e componentes manipulam informações do usuário.
- A aplicação pode fazer uso de padrões de desenvolvimento comuns na indústria.

Abstração:
- Ignorar, em um primeiro momento, o layout da aplicação.

Algoritmos:
- Escrever o código de cada parte do sistema, de forma que os subproblemas sejam de fato resolvidos.
